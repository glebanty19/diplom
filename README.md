# Apple Device Recommender

Метод інтелектуального підбору персоналізованих технічних рішень Apple на основі семантичного аналізу природномовних запитів користувачів.

## Що робить система

Система приймає текстовий запит користувача, наприклад:

```text
ноутбук для програмування
```

Після обробки запиту вона повертає три найбільш релевантні пристрої Apple з показником впевненості та коротким поясненням причин вибору.

## Основна ідея методу

Рекомендація формується на основі гібридного ранжування, яке поєднує чотири компоненти:

- ML-компонент `TF-IDF + Logistic Regression`;
- лексичне IDF-зважене зіставлення токенів;
- rule-based оцінювання технічної відповідності;
- латентно-семантичний аналіз `LSA` на основі `TruncatedSVD + Normalizer`.

Ваги компонентів змінюються адаптивно залежно від структурованості запиту. Якщо користувач явно задає категорію, бюджет, розмір або сценарій використання, більшу роль відіграє rule-based компонент. Для коротких або менш формалізованих запитів підсилюється внесок ML-компонента.

## Файли проєкту

- `app.py` — основна логіка методу, Flask API, нормалізація запиту, виділення намірів, rule-based оцінка, LSA-схожість і фінальне гібридне ранжування.
- `ml_ranker.py` — навчання статистичного класифікатора на синтетично згенерованих прикладах.
- `dataset.py` — база знань пристроїв Apple.
- `evaluate_method.py` — сценарії тестування та перевірка якості рекомендацій.

## Запуск

Встановити потрібні бібліотеки:

```bash
pip install flask flask-cors rapidfuzz numpy scipy scikit-learn
```

Запустити вебзастосунок:

```bash
python app.py
```

Після запуску система доступна за адресою:

```text
http://127.0.0.1:5000/
```

Приклад API-запиту:

```text
http://127.0.0.1:5000/recommend?query=ноутбук для програмування
```

## Тестування та оцінка якості

Система містить розгалужену інфраструктуру для автоматизованого тестування та оцінки якості рекомендацій. Основний інструмент — скрипт `evaluate_method.py`, який реалізує методологію контрольних прикладів (Gold Standard).

### Приклад сценарію тестування (Gold Standard)

Кожен тест-кейс (`EvalCase`) у системі валідується за наступною структурою:

- **Запит**: "недорогий телефон з хорошою камерою"
- **Очікувана категорія**: `iPhone`
- **Бажані результати (Preferred)**: моделі з приставкою Pro минулих поколінь (наприклад, `iPhone 13 Pro`) або актуальні базові моделі (`iPhone 15`).
- **Заборонені результати (Forbidden)**: найдорожчі флагмани (`iPhone 17 Pro Max`) або застарілі моделі з поганою камерою (`iPhone 7`).

Такий підхід дозволяє гарантувати, що гібридна модель ранжування правильно балансує між технічними характеристиками та семантикою запиту.

### Запуск тестів

Для запуску повного циклу тестування виконайте:

```bash
python evaluate_method.py
```

Скрипт імітує реальні HTTP-запити до Flask API, аналізує відповіді та виводить детальний звіт по кожному кейсу.

### Критерії успішності

Тест вважається пройденим (`PASS`), якщо одночасно виконуються три умови:
1. Всі 3 рекомендовані пристрої належать до правильної категорії.
2. Хоча б один пристрій зі списку "Preferred" потрапив у топ-3.
3. Жоден пристрій зі списку "Forbidden" не потрапив у рекомендації.
